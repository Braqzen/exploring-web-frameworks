# Contributing

## How to contribute

1. Read the repository documentation to understand the purpose of this project
2. Create an issue arguing your case and wait for a signal to proceed with your desired work or you may submit a PR that gets rejected
3. Fork, make changes in a branch and open a PR to `master`
4. Link the issue in the PR description and bullet point the changes with reasoning behind them
5. Even if your PR is perfect and seemingly non-ai I will ask questions instead of blindly merging

## How to get rejected

1. Not following points above
2. Low effort
   1. If you cannot be bothered to carefully explain and document your changes then do not expect others to spend their limited life-time on you
3. AI Slop (you will be rejected and blocked/banned)
   1. No vibecoding
   2. You must be able to explain **proficiently**, in your own words and without the use of AI, every single line that has been submitted otherwise the assumption is generic AI Slop submission
4. Do not add new tools, packages, dependencies, any lock files especially if you are the author
   1. I'm not going to spend time investigating if there's something malicious
   2. I _might_ accept exceptionally popular tools but the version needs to be at least a week old from last release with no exploit news

## What to contribute

While this is being built as a public resource this is also a learning experience for me and it's the type of thing I like to do for fun. This means that most contributions are likely to be rejected so it's important to raise an issue so you do not waste your time or mine.

The most likely contribution to be accepted is from someone who is proficient in a framework and can **take an existing implementation from this project** and make it more idiomatic and to a production/professional standard.

You may improve existing code, you may not add new code e.g. if you see that the project does not have framework X and you PR a perfectly professional implementation then you will still be rejected because it takes away from me learning through the pain of whatever mistakes I did not get to make.
